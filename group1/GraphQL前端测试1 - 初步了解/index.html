GraphQL 一种用于 API 的查询语言

GraphQL 既是一种用于 API 的查询语言也是一个满足你数据查询的运行时。 GraphQL 对你的 API 中的数据提供了一套易于理解的完整描述，使得客户端能够准确地获得它需要的数据，而且没有任何冗余，也让 API 更容易地随着时间推移而演进，还能用于构建强大的开发者工具。

二、测试接口 测试地址：http://114.116.100.206:5006/GraphQL/

打开上面的网址，开启的是一个GraphQL 测试工具 ： GraphiQL ，方便开发人员进行测试。当然你可以使用postman,最新版本的postman已经支持GraphQL的调试

1、接口1 查询一个选手，接口名称 player ,输入参数id

query{
  player(id:11)
  {
    id,
    name
  }
}

返回结果

{
  "data": {
    "player": {
      "id": 11,
      "name": "Alex Ovechkin"
    }
  }
}

2、接口2 随机获得一个选手信息，接口名称 random

query{
  random
  {
    id,
    name,
    birthPlace
  }
}

返回结果

{
  "data": {
    "random": {
      "id": 12,
      "name": "Nico Hischier",
      "birthPlace": "Brig, Switzerland"
    }
  }
}

3、接口3 查询所有选手信息，接口名称 all

query{
  all
  {

    name,

    birthPlace

}

返回结果

{

以上示例中字段信息可以根据实际需要进行增加和删除

三、解释

要执行查询，需要有一个根查询对象（query）。查询应该只获取数据，而不应该修改数据。

query {
  player{
    id
    name
  }
}

如果后台只有一个查询对象，可以采用简化的语法

player{
    id
    name
  }

如果需要输入参数，需要写成 (id:12) ，标识 ID = 12 的过滤条件

player(id:12){
    id
    name

}
