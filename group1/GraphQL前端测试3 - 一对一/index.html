假设有两个对象，一个是照片，另外一个是用户。 照片是由用户发布的，每个照片都对应一个用户。

用户表 Users

序号 名 类型 解释
1 Id int 自增ID
2 Name nvarchar(MAX) 姓名
3 Avatar nvarchar(MAX) 头像

照片表 Photos

序号 名 类型 解释
1 Id int 自增ID
2 Name nvarchar(MAX) 姓名
3 Url nvarchar(MAX) 地址
4 Description nvarchar(MAX) 描述
5 created datetime2 创建日期
6 postedBy_UserId int 发布人 关联字段 （1对1）

关系型数据库中定义的结构和前台需要的结构还是有区别的，下面定义了需要的 Schema

type XUser
{
  id:ID!
  name:String
  avatar:String
}
type XPhoto
{
  id:ID!
  name:String!
  url:String!
  description:String!
  created:DateTime!
  postedBy:User!
}

测试用例，输入如下信息获取照片信息及发布人的信息

query{
  photo(id:5){
    id,
    name,
    url,
    postedBy {
      name,
      avatar
    }
  }
}
